## 1. 컬러

페이퍼 크림 + 딥 포레스트 그린의 차분한 베이스에, 테라코타로 강조점을 찍는 시스템입니다.

| Role | Name | Hex | Usage |
|---|---|---|---|
| `--paper` | Paper Cream | `#F5F1EA` | 페이지 배경, 라이트 모드 기본 |
| `--paper-2` | Paper Warm | `#ECE7DA` | 카드/세컨더리 배경, 헤더 strip |
| `--ink` | Ink | `#1C2520` | 본문 텍스트, 워드마크 |
| `--green` | Forest | `#2A3D2E` | 다크 모드 배경, 다크 surface, 깊은 강조 |
| `--green-2` | Forest Light | `#3D5942` | Forest 위 hover/divider |
| `--terra` | **Terra (Accent)** | **`#C4633B`** | 마크, 워드마크 언더라인, 링크, 카테고리 라벨 |
| `--terra-soft` | Terra Soft | `#D98F6A` | Forest 배경 위 마크/액센트 (가독성 확보용) |
| `--mute` | Mute | `rgba(28,37,32,0.55)` | 보조 텍스트, 메타데이터 |
| `--rule` | Rule | `rgba(28,37,32,0.18)` | 구분선 |

### CSS Variables (drop-in)

```css
:root {
  --paper: #f5f1ea;
  --paper-2: #ece7da;
  --ink: #1c2520;
  --green: #2a3d2e;
  --green-2: #3d5942;
  --terra: #c4633b;
  --terra-soft: #d98f6a;
  --mute: rgba(28, 37, 32, 0.55);
  --rule: rgba(28, 37, 32, 0.18);
}

/* Dark mode — Forest 베이스로 반전 */
@media (prefers-color-scheme: dark) {
  :root {
    --paper: #2a3d2e;
    --paper-2: #1f2e22;
    --ink: #f0ecdf;
    --terra: #d98f6a;   /* terra-soft를 다크에서 primary로 사용 */
    --mute: rgba(240, 236, 223, 0.55);
    --rule: rgba(240, 236, 223, 0.18);
  }
}
```

### 사용 규칙

- **Terra**(`#C4633B`)는 한 페이지에서 **5–10%** 이하로만 등장. 마크·언더라인·카테고리 라벨·CTA 정도.
- 본문 위 강조는 `border-bottom: 1.5px solid var(--terra)` 또는 `color: var(--terra)` — 형광펜처럼 면으로 칠하지 말 것.
- Forest Green은 다크 모드 surface 또는 인용 카드 같은 weighted 영역에만. 페이지 배경으로는 사용하지 않음 (라이트 모드 기준).

## 2. 타이포그래피

### 브랜드 잠금 (필수)

워드마크는 **반드시 Nanum Gothic Coding** (한글 모노스페이스)를 사용합니다.

```css
/* Brand font (워드마크 전용) */
@import url('https://fonts.googleapis.com/css2?family=Nanum+Gothic+Coding:wght@400;700&display=swap');

.brand-wordmark {
  font-family: 'Nanum Gothic Coding', monospace;
  font-weight: 700;
  letter-spacing: -0.02em;
  line-height: 1;
}
```

라틴 보조용으로 **JetBrains Mono**를 사용 (메타데이터·카테고리 라벨·N°.001 같은 숫자 표기):

```css
@import url('https://fonts.googleapis.com/css2?family=JetBrains+Mono:ital,wght@0,400;0,500;0,600;0,700;1,400&display=swap');

.brand-meta {
  font-family: 'JetBrains Mono', monospace;
  letter-spacing: 0.22em;       /* 0.18–0.32 사이에서 사이즈에 맞게 */
  text-transform: uppercase;
}
```

### 본문 폰트 (자유 선택)

본문은 **별도로 선택 예정**. 추천:
- **Pretendard Variable** — 한글 + 라틴 모두 모던하고 가독성 우수 (가장 안전한 선택)
- **Noto Sans KR** — 보편적, Google Fonts CDN
- **IBM Plex Sans KR** — 살짝 인더스트리얼한 결, 워드마크의 모노 톤과 친화적

본문 사이드에서 사용하는 css 변수만 분리해두면 교체가 쉽습니다:

```css
:root {
  --font-brand: 'Nanum Gothic Coding', monospace;   /* 워드마크 — 고정 */
  --font-meta:  'JetBrains Mono', monospace;        /* 메타/라벨 — 고정 */
  --font-body:  'Pretendard Variable', sans-serif;  /* 본문 — TBD */
}
```

## 3. 마크 (Solid Comma Spiral)

안으로 말려 들어가는 솔리드 면. 코칭의 "안으로 들어가는 1:1 대화"와 노트의 손그림 도형이 만난 형태입니다.

### Inline SVG (React/HTML 어디든 그대로 사용)

```html
<svg viewBox="0 0 220 220" xmlns="http://www.w3.org/2000/svg" aria-label="코칭 노트">
  <path
    d="M 110 22 C 165 22, 198 60, 198 110 C 198 162, 158 198, 110 198
       C 62 198, 22 162, 22 114 C 22 78, 50 50, 86 50
       C 116 50, 140 72, 140 102 C 140 126, 122 144, 100 144
       C 84 144, 72 132, 72 116 C 72 106, 80 98, 90 98
       C 96 98, 102 102, 104 108"
    fill="#C4633B" stroke="#C4633B" stroke-width="2" stroke-linejoin="round"/>
</svg>
```

`fill`/`stroke`를 `currentColor`로 바꾸면 부모의 `color`를 따라가도록 만들 수 있습니다 — 다크 모드 토글에 유용:

```html
<svg viewBox="0 0 220 220" style="color: var(--terra)">
  <path d="…같은 path…" fill="currentColor" stroke="currentColor" stroke-width="2"/>
</svg>
```

### React 컴포넌트

```jsx
export function SpiralMark({ size = 26, color = 'var(--terra)' }) {
  return (
    <svg
      viewBox="0 0 220 220"

      style={{ display: 'block' }}
    >
      <path
        d="M 110 22 C 165 22, 198 60, 198 110 C 198 162, 158 198, 110 198

           C 116 50, 140 72, 140 102 C 140 126, 122 144, 100 144
           C 84 144, 72 132, 72 116 C 72 106, 80 98, 90 98
           C 96 98, 102 102, 104 108"
        fill={color}
        stroke={color}
        strokeWidth="2"
        strokeLinejoin="round"
      />
    </svg>
  );
}
```

### 파일 에셋 (이 폴더에 함께 포함)

| 파일 | 색상 | 용도 |
|---|---|---|
| `mark.svg` | `#C4633B` Terra | 기본 마크 |
| `mark-cream.svg` | `#F5F1EA` | 다크 배경 위 — Forest surface 등 |
| `mark-ink.svg` | `#1C2520` | 단색 인쇄·도장·워터마크 |
| `favicon.svg` | `#C4633B` | 라이트 파비콘 |
| `favicon-dark.svg` | Terra-soft on Forest | 다크 모드 파비콘 |

### 클리어 스페이스
마크 주변 모든 면에 **마크 높이의 0.25×** 이상의 여백을 확보. 다른 요소가 그 안에 들어오지 않도록.

### 최소 사이즈
- 디지털: **16px** (파비콘 한계)
- 인쇄: **8mm**

## 4. 워드마크

### 구조
"코칭 노트" — Nanum Gothic Coding 700 + 한글 글자 아래 손그림 Terra 언더라인.

언더라인은 **하드코딩된 SVG 곡선**이며, 글자에 정렬되도록 워드마크 사이즈에 비례합니다.

### React 컴포넌트 (canonical)

```jsx
export function Wordmark({
  size = 22,
  dark = false,
  accent = 'var(--terra)',
}) {
  const width = size * 4.6;
  const height = size * 0.22;
  const stroke = Math.max(1.6, size * 0.06);

  return (
    <div style={{ position: 'relative', padding: '4px 4px', display: 'inline-block' }}>
      <div
        style={{
          fontFamily: "'Nanum Gothic Coding', monospace",
          fontSize: size,
          fontWeight: 700,
          letterSpacing: '-0.02em',
          lineHeight: 1,
          color: dark ? '#f0ecdf' : 'var(--ink)',
          position: 'relative',
          zIndex: 1,
          whiteSpace: 'nowrap',
        }}
      >
        코칭 노트
      </div>
      <svg
        width={width}
        height={height}
        style={{ position: 'absolute', left: 0, bottom: -size * 0.04 }}
        aria-hidden="true"
      >
        <path
          d={`M2 ${size * 0.13} Q ${size * 1.1} ${size * 0.03}, ${size * 2.2} ${size * 0.11} T ${size * 4.5} ${size * 0.09}`}
          stroke={accent}
          strokeWidth={stroke}
          fill="none"
          strokeLinecap="round"
        />
      </svg>
    </div>
  );
}
```

### 색상 변형
- **라이트 위**: 글자 `--ink`, 언더라인 `--terra`
- **다크/Forest 위**: 글자 `#F0ECDF`, 언더라인 `--terra-soft` (`#D98F6A`) — 가독성 ↑

## 5. 락업 (Mark + Wordmark)

### 표준 비율 (반드시 이걸 따를 것)

> **마크 사이즈 ≈ 워드마크 font-size × 1.18**
> **gap ≈ 마크 사이즈의 0.3×**

가장 많은 시안에서 균형이 잡혔던 비율입니다. (Before/After 비교 결과)

| Size | Mark | Wordmark | Gap | 사용처 |
|---|---|---|---|---|
| **Small** | 17px | 14px | 5px | 푸터, 본문 내 inline 브랜드 |
| **Medium** | 26px | 22px | 8px | **데스크톱 nav · 표준** |
| **Large** | 42px | 36px | 12px | 명함 헤더, 히어로 |

### 코드 예시

```jsx
import { SpiralMark, Wordmark } from './brand';

// Medium (표준)
<header style={{ display: 'flex', alignItems: 'center', gap: 8 }}>
  <SpiralMark size={26} />
  <Wordmark size={22} />
</header>
```

### 절대 하지 말 것
- 마크와 워드마크 사이에 다른 요소를 끼우지 말 것 (구분선·아이콘·텍스트 X)
- 마크를 워드마크보다 작게 만들지 말 것 (1:1 비율 ✕ — 평평해 보임)
- 마크를 워드마크 font-size의 1.4× 이상으로 키우지 말 것 (무거워 보임)

## 6. 파비콘

`mark.svg`를 그대로 favicon으로 사용.

```html
<!-- 기본 (모든 모던 브라우저) -->
<link rel="icon" type="image/svg+xml" href="/favicon.svg">

<!-- 다크 모드 자동 전환 (선호) -->
<link rel="icon" type="image/svg+xml" href="/favicon.svg"
      media="(prefers-color-scheme: light)">
<link rel="icon" type="image/svg+xml" href="/favicon-dark.svg"
      media="(prefers-color-scheme: dark)">

<!-- 폴백 (구형 Safari 등) -->
<link rel="icon" type="image/png" sizes="32x32" href="/favicon-32.png">
<link rel="apple-touch-icon" href="/apple-touch-icon.png">
```

### iOS Apple Touch Icon 만들기
180×180 PNG, 양쪽 패딩 ≈ 12%, 배경은 `#F5F1EA` (Paper Cream)로 채우고 가운데 마크.

### Favicon 크기별 권장
- `16×16`, `32×32`, `48×48` PNG 폴백 — 솔리드 형태라 작은 사이즈에서도 깨지지 않음
- `180×180` Apple Touch (배경 추가, 마크 76% 영역)

## 10. 부록 — 디자인 결정 히스토리

이 시스템이 어떻게 정해졌는지 (검수/롤백 시 참조):

- **워드마크**: 10가지 시안 중 `04 Marked Up` 선정 — 손글씨 체크 + 곡선 언더라인. "관찰·기록"이라는 코칭의 핵심을 담음.
- **마크**: 추상 5종 → 그중 Spiral 방향 채택 → Spiral 4변형 중 `03 Solid Comma` 선정 — 작은 사이즈(16px 파비콘)에서도 살아남는 솔리드 면.
- **락업 비율**: Before(1.45×) / After(1.0×) / Middle(1.18×) 비교 후 Middle 채택.
- **포인트 컬러**: A 테라코타 / B 버건디 / C 슬레이트 블루 중 **A 테라코타** 선정 — 따뜻함 · 손글씨와의 친화성 · 코칭 카테고리에서의 차별성.
- **본문 폰트**: 구현 단계에서 별도 선택 (Pretendard / Noto Sans KR / IBM Plex Sans KR 등 중에서).
